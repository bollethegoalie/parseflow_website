API Reference

Complete guide to the public request contract.

Authentication​

Public usage is through your Parseflow API key:

  • X-API-Key

For advanced BYOK requests, use X-BYOK-Key and mode=byok_assisted.

POST /process​

Legacy-compatible parse-and-chunk endpoint.

  • mode=basic is accepted as a compatibility alias.
  • mode=semantic-lite and other chunking modes remain accepted where the backend allows them.

POST /v2/process​

Structured extraction pipeline.

Inputs​

NameTypeRequiredNotes
filefilenoPDF, DOCX, or text file
textstringnoRaw text input
chunk_sizeintnoDefault 2000
overlapintnoDefault 200
modestringnodeterministic or byok_assisted; legacy chunking aliases are accepted as compatibility inputs
presetstringnorag-optimized, summarization-friendly, extraction-focused
schema_jsonstringnoJSON schema string
enforce_schemaboolnoReturns 422 if required schema fields fail
byok_providerstringnoRequired when mode=byok_assisted
byok_modelstringnoRequired when mode=byok_assisted
X-BYOK-KeyheadernoRequired when mode=byok_assisted

Response fields​

  • api_version
  • request_id
  • document
  • result
  • metadata_summary
  • preset
  • extraction
  • analysis
  • costs

Example​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/process \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "Idempotency-Key: req-123" \
  -F "text=Company: Acme Inc\nDate: 2025-01-15\nAmount: $5000" \
  -F "mode=deterministic" \
  -F "preset=extraction-focused" \
  -F "schema_json={\"fields\":[{\"name\":\"company\",\"required\":true},{\"name\":\"date\",\"type\":\"date\"},{\"name\":\"amount\"}]}"

POST /v2/analyze​

Returns the analysis payload without the full extraction envelope.

GET /v2/stats​

Returns usage totals globally or for the current client label.

POST /v2/jobs​

Creates an async job. This is a paid-lane route.

POST /v2/batch​

Processes multiple documents in one request. This is a paid-lane route.

POST /v2/index​

Indexes a document for later search.

GET /v2/search?q=...​

Searches indexed documents.

Limits and validation​

  • Deterministic traffic uses your Parseflow API key.
  • Async and batch are part of paid usage.
  • BYOK advanced requests require BYOK headers.
  • Too large input returns 413.
  • Invalid request data returns 400.
  • Invalid schema JSON returns 400.
  • Schema validation failures can return 422.
  • Timeout returns 504.
