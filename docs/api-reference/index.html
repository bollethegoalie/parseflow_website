API Reference

Complete guide to all public endpoints, headers, request bodies, response shape, and launch-specific environment variables.

Authentication​

The backend uses two layers of protection:

  • X-Docflow-Gateway-Secret when DOCFLOW_GATEWAY_SECRET is configured
  • X-API-Key when DOCFLOW_REQUIRE_API_KEY=true

RapidAPI handles its own public key flow, but the same backend still accepts the gateway secret if your deployment requires it.

Header names​

HeaderPurpose
X-Docflow-Gateway-SecretOptional gateway auth
X-API-KeySigned API key for v2 endpoints
X-Docflow-Admin-SecretIssues new API keys
X-Client-IdUsage tracking bucket
Idempotency-KeyCached v2 process responses

POST /process​

Basic parse-and-chunk endpoint.

Inputs: file or text, plus chunk_size, overlap, and mode. https://docflow-xvmh.onrender.com/process Output: request_id, document, result, metadata_summary. https://docflow.p.rapidapi.com/process

Example​

curl -X POST http://127.0.0.1:8000/process \
  -H "X-Docflow-Gateway-Secret: your-secret" \
  -F "text=Simple document" \
  -F "chunk_size=1200" \
  -F "mode=basic"

POST /pack​

Same processing flow as /process, but the response is a ZIP download.

The archive contains:

  1. results.json
  2. document.md
  3. chunks.jsonl

Example​

curl -X POST http://127.0.0.1:8000/pack \
  -H "X-Docflow-Gateway-Secret: your-secret" \
  -F "file=@sample.pdf" \
  -F "zip_name=sample-pack.zip" \
  -o sample-pack.zip

POST /v2/process​

Structured extraction pipeline.

Inputs​

NameTypeRequiredNotes
filefilenoPDF, DOCX, or text file
textstringnoRaw text input
chunk_sizeintnoDefault 2000
overlapintnoDefault 200
modestringnobasic or semantic-lite
presetstringnorag-optimized, summarization-friendly, extraction-focused
schema_jsonstringnoJSON schema string
enforce_schemaboolnoReturns 422 if required schema fields fail

Headers​

HeaderRequiredNotes
X-API-KeynoRequired when API key enforcement is on
X-Client-IdnoUsage tracking label
Idempotency-KeynoReturns cached response on repeat requests
X-Docflow-Gateway-SecretmaybeRequired if your deployment config sets DOCFLOW_GATEWAY_SECRET

Response fields​

  • api_version
  • request_id
  • document
  • result
  • metadata_summary
  • preset
  • extraction
  • analysis

Example​

curl -X POST http://127.0.0.1:8000/v2/process \
  -H "Idempotency-Key: req-123" \
  -H "X-Client-Id: my-app" \
  -F "text=Company: Acme Inc\nDate: 2025-01-15\nAmount: $5000" \
  -F "preset=extraction-focused" \
  -F "schema_json={\"fields\":[{\"name\":\"company\",\"required\":true},{\"name\":\"date\",\"type\":\"date\"},{\"name\":\"amount\"}]}"

POST /v2/analyze​

Returns the analysis payload without the full extraction envelope.

Useful for benchmark comparisons and cost estimation.

GET /v2/stats​

Returns usage totals globally or for the current client label.

POST /v2/admin/api-keys​

Issues a signed API key.

Headers:

HeaderPurpose
X-Docflow-Admin-SecretRequired admin secret

Form fields:

NameTypeNotes
labelstringOptional human-readable label

POST /v2/jobs​

Creates an async job.

Form fields:

NameTypeNotes
filefileOptional
textstringOptional
presetstringOptional
callback_urlstringOptional webhook URL

GET /v2/jobs/{job_id}​

Poll job status and retrieve the final result once completed.

POST /v2/jobs/{job_id}/deliver​

Simulates webhook delivery for local testing.

POST /v2/batch​

Processes multiple documents in one request.

Example body​

{
  "documents": [
    {"text": "Invoice #1 due 2024-01-01"},
    {"text": "Contract agreement effective 2024-02-01"}
  ]
}

Limits and validation​

The current service config defaults to:

  • 10 MB max upload size
  • 100 max chunks
  • 10 second processing timeout
  • basic and semantic-lite modes

Invalid or missing inputs return structured JSON errors such as:

{
  "detail": {
    "code": "MISSING_INPUT",
    "message": "provide a file or text"
  }
}

Environment variables to know​

VariablePurpose
DOCFLOW_MAX_UPLOAD_BYTESMax upload size
DOCFLOW_MAX_CHUNKSMaximum chunk count
DOCFLOW_ALLOWED_MODESAllowed chunking modes
DOCFLOW_PROCESSING_TIMEOUT_SECONDSPer-request timeout
DOCFLOW_GATEWAY_SECRETGateway auth secret
DOCFLOW_API_KEY_SECRETAPI key signing secret
DOCFLOW_API_KEY_ADMIN_SECRETAdmin key issuance secret
DOCFLOW_REQUIRE_API_KEYRequire API key on v2 routes
DOCFLOW_ALLOW_EXTERNAL_DELIVERYAllow real webhook delivery
DOCFLOW_WEBHOOK_SECRETSign webhook payloads
  1. Start with mode=basic — Fast, no surprises
  2. Use chunk_size=2000 — Good default for most LLMs
  3. Check tokens_estimate — Know your costs before feeding to LLM
  4. Batch requests — Don't send 100 files one-at-a-time
  5. Store request_id — Useful for debugging
