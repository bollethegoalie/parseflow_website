Model Cost Comparison

This page compares Parseflow's public lanes with current flagship model pricing from OpenAI, Anthropic, and Google.

The goal is not to claim a fixed savings number for every workflow. The goal is to show where Parseflow can save money: by reducing how much raw document text you send to an expensive model, and by letting deterministic extraction handle requests that do not need a large model at all.

Parseflow lane recap​

From the public docs:

  • Free deterministic lane: $0, 50 requests per month, 1 request per minute, 1 MB max upload.
  • Starter paid lane: 2,000 requests per month, 20 requests per minute, 10 MB max upload, async jobs, and batch processing.
  • BYOK advanced lane: advanced extraction through byok_assisted, with your own provider key.

Parseflow's public docs do not currently publish a per-request dollar price for the paid lane, so this page focuses on where Parseflow can reduce downstream model spend.

Example: raw document vs Parseflow-prepared text​

Imagine a large contract or invoice packet that you would otherwise send directly to a flagship model.

Example workload:

  • Raw document prompt: 100,000 input tokens
  • Direct model output: 20,000 output tokens
  • Parseflow-prepared version: 10,000 input tokens and 2,000 output tokens after chunking, extraction, and trimming the prompt to only the useful text

That is not a universal ratio, but it is a reasonable illustration of why Parseflow can lower spend.

What Parseflow saves you money on​

Parseflow is most useful when you would otherwise:

  • Send a raw document to a model just to find the relevant parts.
  • Repeatedly ask the model to re-read the same long text.
  • Pay for expensive output tokens when a smaller, structured extraction would work.
  • Use a flagship model for tasks that a deterministic pass can handle first.

That means the savings usually come from lower token usage and fewer unnecessary model calls, not from Parseflow magically making provider pricing disappear.

When BYOK makes sense​

BYOK is different from the paid lane:

  • Parseflow still runs the built-in advanced extraction route.
  • You provide the model key, so the provider bill goes to you.
  • Parseflow avoids subsidizing inference costs.

That is useful when a team already has provider access or wants to keep model billing on their own account.

Simple rule of thumb​

  • If you only need document automation, deterministic mode is the cheapest path.
  • If you need advanced extraction and want Parseflow to manage the model side, use the paid lane.
  • If you already have provider credits or want to control the model bill yourself, use BYOK.
