Tutorials

Step-by-step guides for common Parseflow workflows. Each tutorial includes a real-world scenario and shows you exactly what to do.

Tutorial 1: Extract Invoice Data​

Scenario: You receive invoices as PDFs and need to extract key financial data (invoice number, date, amount, due date) to feed into your accounting system.

Step 1: Get your API key​

  1. Visit Billing
  2. Click "Buy Starter Plan"
  3. Complete Stripe checkout
  4. Copy your API key from your email

Step 2: Prepare your invoice file​

Save your PDF invoice as invoice.pdf in a working directory.

Step 3: Send the request​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/process \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-Client-Id: my-accounting-system" \
  -F "file=@invoice.pdf" \
  -F "mode=deterministic" \
  -F "preset=extraction-focused" \
  -F "schema_json={
    \"fields\": [
      {\"name\": \"invoice_number\", \"type\": \"string\", \"required\": true},
      {\"name\": \"date\", \"type\": \"date\", \"required\": true},
      {\"name\": \"due_date\", \"type\": \"date\", \"required\": true},
      {\"name\": \"amount\", \"type\": \"number\", \"required\": true},
      {\"name\": \"vendor_name\", \"type\": \"string\"},
      {\"name\": \"line_items\", \"type\": \"array\"}
    ]
  }"

Step 4: Parse the response​

The response will include:

{
  "extraction": {
    "fields": {
      "invoice_number": {
        "value": "INV-2024-001",
        "confidence": 0.98
      },
      "date": {
        "value": "2024-02-15",
        "confidence": 0.95
      },
      "amount": {
        "value": 5000.00,
        "confidence": 0.97
      }
    }
  }
}

Step 5: Use the data​

Store each extracted field in your accounting system:

# Pseudocode: Using extracted data
import requests
import json

api_key = "YOUR_PARSEFLOW_API_KEY"
files = {"file": open("invoice.pdf", "rb")}
headers = {"X-API-Key": api_key, "X-Client-Id": "my-system"}

response = requests.post(
  "https://jsonparser-production-f9af.up.railway.app/v2/process",
  files=files,
  headers=headers,
  data={
    "preset": "extraction-focused",
    "schema_json": '{"fields": [{"name": "invoice_number"}, {"name": "amount"}]}'
  }
)

data = response.json()
invoice_number = data["extraction"]["fields"]["invoice_number"]["value"]
amount = data["extraction"]["fields"]["amount"]["value"]

# Send to your accounting system
accounting_system.record_invoice(invoice_number, amount)

Tutorial 2: Batch Process Multiple Documents​

Scenario: You have 50 receipts and want to extract amounts from all of them in one request instead of making 50 separate API calls.

Step 1: Prepare your batch​

Create a JSON file called batch-request.json:

{
  "documents": [
    {
      "text": "Receipt #001\nDate: 2024-02-15\nTotal: $45.50\nVendor: Coffee Shop"
    },
    {
      "text": "Receipt #002\nDate: 2024-02-15\nTotal: $120.00\nVendor: Restaurant"
    },
    {
      "text": "Receipt #003\nDate: 2024-02-16\nTotal: $35.75\nVendor: Gas Station"
    }
  ]
}

Step 2: Send the batch request​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/batch \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "Content-Type: application/json" \
  -d @batch-request.json

Step 3: Process the results​

Response contains results for all documents:

{
  "results": [
    {
      "document_index": 0,
      "status": "success",
      "extraction": {
        "fields": {
          "total": {"value": 45.50, "confidence": 0.98}
        }
      }
    },
    {
      "document_index": 1,
      "status": "success",
      "extraction": {
        "fields": {
          "total": {"value": 120.00, "confidence": 0.99}
        }
      }
    }
  ]
}

Tutorial 3: Index Documents for Search​

Scenario: You're building a document knowledge base and want to let users search through uploaded contracts by keyword.

Step 1: Add documents to the index​

# Index first contract
curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/index \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-Client-Id: contract-search" \
  -F "text=Service Agreement between Acme Inc and Widgets Corp. Payment terms: Net 30. Effective date: 2024-02-01" \
  -F "document_id=contract-2024-001" \
  -F "metadata={\"client\": \"Acme Inc\", \"year\": 2024}"

# Index second contract
curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/index \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-Client-Id: contract-search" \
  -F "text=Maintenance Agreement between TechCorp and WidgetMakers. Payment terms: Monthly. Effective: 2024-01-15" \
  -F "document_id=contract-2024-002" \
  -F "metadata={\"client\": \"TechCorp\", \"year\": 2024}"

Step 2: Search the index​

User searches for "Payment terms":

curl -G https://jsonparser-production-f9af.up.railway.app/v2/search \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  --data-urlencode "q=Payment terms" \
  --data-urlencode "limit=10"

Step 3: Display search results​

Response:

{
  "results": [
    {
      "document_id": "contract-2024-001",
      "relevance_score": 0.95,
      "snippet": "...Payment terms: Net 30...",
      "metadata": {"client": "Acme Inc"}
    },
    {
      "document_id": "contract-2024-002",
      "relevance_score": 0.87,
      "snippet": "...Payment terms: Monthly...",
      "metadata": {"client": "TechCorp"}
    }
  ]
}

Tutorial 4: Async Processing with Webhooks​

Scenario: You have very large documents (50+ MB) that take time to process. You don't want to wait for the response; instead, you want Parseflow to send results to your webhook URL when ready.

Step 1: Set up your webhook endpoint​

Your backend needs an endpoint that accepts POST requests:

# Flask example
from flask import Flask, request

app = Flask(__name__)

@app.route('/parseflow-webhook', methods=['POST'])
def handle_parseflow_result():
    data = request.json
    job_id = data.get('job_id')
    result = data.get('result')
    
    # Store result in database
    db.store_parsing_result(job_id, result)
    
    # Trigger downstream processing
    send_to_analytics(result)
    
    return {"status": "received"}, 200

if __name__ == '__main__':
    app.run()

Step 2: Submit an async job​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/jobs \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-Client-Id: large-doc-processor" \
  -F "file=@very_large_document.docx" \
  -F "preset=rag-optimized" \
  -F "callback_url=https://myapp.com/parseflow-webhook"

Step 3: Receive the webhook​

When processing completes, Parseflow sends:

{
  "job_id": "job-abc123def456",
  "status": "completed",
  "result": {
    "document": {
      "filename": "very_large_document.docx",
      "size_bytes": 52428800
    },
    "result": {
      "text": "...",
      "chunks": [...]
    }
  }
}

Step 4: Handle the result​

Your webhook processes the data and continues the workflow without the user waiting.

Tutorial 5: Using BYOK (Bring Your Own Key)​

Scenario: You already have an OpenAI API key and want to use advanced extraction with your own model, keeping the inference bill on your own account.

Step 1: Get your OpenAI API key​

  1. Visit OpenAI API keys
  2. Create a new secret key
  3. Copy it somewhere safe

Step 2: Prepare your document​

# You can use text or a file
TEXT_CONTENT="Company Name: TechCorp Inc\nAddress: 123 Main St, San Francisco, CA\nPhone: (415) 555-0123\nWebsite: techcorp.com"

Step 3: Send BYOK request​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/process \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-BYOK-Key: sk-YOUR_OPENAI_KEY" \
  -F "text=$TEXT_CONTENT" \
  -F "mode=byok_assisted" \
  -F "byok_provider=openai" \
  -F "byok_model=gpt-4o-mini" \
  -F "schema_json={
    \"fields\": [
      {\"name\": \"company_name\", \"type\": \"string\"},
      {\"name\": \"address\", \"type\": \"string\"},
      {\"name\": \"phone\", \"type\": \"string\"},
      {\"name\": \"website\", \"type\": \"string\"}
    ]
  }"

Step 4: Result comes from your model​

{
  "extraction": {
    "fields": {
      "company_name": {
        "value": "TechCorp Inc",
        "confidence": 0.99
      },
      "address": {
        "value": "123 Main St, San Francisco, CA",
        "confidence": 0.98
      }
    }
  }
}

The billing goes to your OpenAI account, not Parseflow's.

Tips for Success​

  1. Start simple: Test with mode=deterministic first before trying BYOK or async
  2. Define schemas precisely: Better schemas = higher confidence scores
  3. Use presets: extraction-focused is fastest, rag-optimized best for chunking
  4. Handle errors: Always check confidence scores and have a manual review fallback
  5. Monitor costs: Track token usage in the response to understand pricing
  6. Use X-Client-Id: Label your requests so you can track usage per client
