FAQ

General Questions​

Q: What is Parseflow?

A: Parseflow is a document automation engine that turns files into structured data. Upload a PDF, Word document, or text file. Get back parsed text, organized chunks, extracted fields, confidence scores, and search-ready metadata.

Q: What file types are supported?

A: PDF, DOCX, TXT, and images (via OCR). Parseflow automatically detects the file type and applies the appropriate parser.

Q: Is Parseflow open source?

A: No. Parseflow is a proprietary product. The public documentation explains API behavior and integration paths; it does not publish the private source code for the product.

Q: How long does processing take?

A: Most documents process in 2-5 seconds. Very large documents (over 50MB) may take 10-30 seconds. Use async jobs for batch processing without blocking.

Q: Can I self-host Parseflow or run it offline?

A: Self-hosting or air-gapped deployments are not part of the public offering. Contact the team for enterprise or private deployment options.

API & Integration Questions​

Q: What is the difference between /process and /v2/process?

A: /process (v1) is simpler and returns basic extraction. /v2/process adds structured field extraction, presets, schema validation, and analysis. Use v2 for production.

Q: How do I authenticate API requests?

A: Use X-API-Key for public requests. For advanced extraction, add X-BYOK-Key.

Q: Can I use Parseflow without an API key?

A: Free evaluation still uses an API key after checkout, but the key is issued automatically and there is no manual delivery step.

Q: What is an Idempotency-Key?

A: A header that prevents duplicate processing if your request is retried. Submit the same Idempotency-Key with a duplicate request and get the cached response from the first request.

Q: How do async jobs work?

A: Submit a document to /v2/jobs. Get back a job ID. Poll /v2/jobs/job-id for status. Optionally provide a callback URL for webhook delivery when complete.

Q: Do you support webhooks?

A: Yes. Async jobs can send results to your callback URL when processing completes.

Q: Can I index documents for search?

A: Yes. Use POST /v2/index to add documents to the search index. Use GET /v2/search?q=keyword to retrieve them later by keyword.

Extraction & Accuracy Questions​

Q: How accurate is field extraction?

A: Typical accuracy is 85-95% for structured documents like invoices. Accuracy depends on document quality, clarity, and how specific your schema is. Test with your actual documents.

Q: What is a confidence score?

A: Each extracted field returns a confidence score from 0 to 1 indicating likelihood of accuracy. Scores above 0.9 indicate very likely correct results. Scores below 0.5 indicate high chance of error.

Q: How do I improve extraction accuracy?

A: Use high-resolution source documents, define a specific schema, use the extraction-focused preset, ensure consistent field placement, and test and iterate.

Q: Can I extract custom fields?

A: Yes, define any JSON schema with field names, types, and descriptions. Parseflow extracts whatever fields your schema specifies.

Q: Does Parseflow support handwritten documents?

A: Limited support. Handwriting recognition requires OCR, which is less accurate than printed text. Handwritten documents typically have 50-70% accuracy.

Q: Can Parseflow read tables?

A: Yes. Parseflow detects and preserves table structures. Returns table data as organized arrays with row/column metadata.

Q: Can Parseflow handle multi-language documents?

A: Yes, English is primary. Other languages supported with lower accuracy. Best results with English or single-language documents.

Chunking & Processing Questions​

Q: What is document chunking?

A: Breaking a large document into smaller, overlapping pieces for processing and search. Chunks are typically 2000 characters with 200-character overlap.

Q: What is the difference between basic and semantic-lite chunking?

A: Basic equals simple character-based splitting. Semantic-lite respects paragraph/heading boundaries, preserves tables, and produces more coherent chunks.

Q: Can I customize chunk size?

A: Yes, chunk_size parameter controls size (default 2000), overlap controls overlap (default 200). Valid ranges: chunk_size 200-10,000, overlap 0-chunk_size-1.

Q: What is a preset?

A: Pre-configured combination of chunk_size, overlap, and mode. Choose from:

  • extraction-focused (best for field extraction)
  • rag-optimized (best for search/RAG)
  • summarization-friendly (best for summaries)

Performance & Limits Questions​

Q: What are the processing time limits?

A: Default timeout is 10 seconds per document. Pro/Business plans offer extended timeouts. Contact sales for custom limits.

Q: How large can files be?

A: Free lane: 1 MB. Starter paid: 10 MB. BYOK advanced requests follow the paid upload cap. Exceeding limits returns 413 Payload Too Large.

Q: How many chunks can I create?

A: Default limit is 100 chunks per document. Exceeding returns 400 error. Contact support for higher limits if needed.

Q: What is your rate limit?

A: Free lane: 1 req/min. Starter paid: 20 req/min. BYOK advanced requests are capped at 5 req/min.

Pricing & Plans Questions​

Q: How much does Parseflow cost?

A: Starter monthly is $9/month. Growth monthly is $24/month. BYOK is the advanced path if you already have your own provider key.

Q: What is included in each plan?

A: Both plans give you Parseflow API access. Growth is the better-value option for more regular usage. Current route limits and behavior are documented on the Limits page.

Q: What is BYOK (Bring Your Own Key)?

A: Use your own model provider key for advanced extraction. It gives you the advanced path without relying on Parseflow-managed inference.

Q: Do you offer discounts?

A: Public launch pricing is kept simple. If pricing changes later, the pricing page will be updated.

Q: What happens when I exceed my plan limit?

A: Requests that exceed route or usage limits return an error response instead of silently continuing. Check the response body and the Limits page.

Data & Security Questions​

Q: Where is my data stored?

Q: How long is data retained?

A: Retention depends on your integration path and whether you index documents for search.

Q: Do you have HIPAA or SOC2 compliance?

A: For compliance questions, contact the team using the contact page.

Q: Can I self-host Parseflow?

A: This public site focuses on the hosted API integration path.

Q: Is data encrypted?

A: Requests are expected to run over HTTPS through the public API gateway.

Troubleshooting Questions​

Q: Why did my processing fail?

A: Common causes: file too large (413 error), invalid schema (422 error), timeout (504 error), unsupported format. Check error message for details.

Q: How do I improve extraction quality?

A: Use high-resolution documents, specific schemas, extraction-focused preset. Test with sample documents first. Iteratively improve schema based on results.

Q: Why are confidence scores low?

A: Document quality issues, ambiguous field definitions, complex formatting. Increase accuracy by improving source document clarity and schema specificity.

Q: Can I get detailed error messages?

A: Yes, all errors include error_code and message. Contact support for additional debugging help.

Q: How do I optimize performance?

A: Use appropriate chunk sizes, async jobs for larger workloads, and search when retrieval is better than reprocessing. Start simple, then tune based on real usage.
