Postman Quickstart

Use Postman to test the live Parseflow API at https://api.parseflow.tech.

1. Import the collection​

Download parseflow-demo-public.postman_collection.json from this site, or grab it from the jsonparser repo.

In Postman: File → Import → Upload the JSON file.

2. Create an environment​

Create an environment named Parseflow Live with:

VariableValue
baseUrlhttps://api.parseflow.tech
API_KEYYour pfk_* key (leave blank until you have one)
API_KEY_HEADER_NAMEX-API-Key

Select Parseflow Live from the environment dropdown before sending requests.

3. Check health​

Run Health (GET {{baseUrl}}/health). Expected response:

{
  "status": "ok",
  "version": "0.2.0"
}

4. Get a trial key (no purchase required)​

Run Request trial key (POST {{baseUrl}}/v2/keys):

{
  "email": "you@example.com",
  "device_id": "postman"
}
  • Use a real email (disposable domains are blocked).
  • When email delivery is configured, the response is "status": "sent" and the key arrives in your inbox.
  • Trial keys expire in 24 hours with a small request cap.
  • Paste the key into your API_KEY environment variable.

For paid plans, checkout via Billing — Whop sends a production pfk_* key by email.

5. Process a document​

Run Process (file upload) or Process (text only) with your API_KEY set.

Optional form fields:

  • mode — deterministic (default) or byok_assisted
  • preset — extraction-focused, rag-optimized, or summarization-friendly

6. Inspect your key​

Run Key Info (GET {{baseUrl}}/v2/me) or Usage (GET {{baseUrl}}/v2/usage) to see plan limits and remaining quota.

Publish your own Postman workspace​

If you maintain the public Postman Documenter link:

  1. Import the updated collection JSON from this repo.
  2. In Postman, open the collection → ⋯ → Share → publish or update the public documentation.
  3. Confirm every request uses {{baseUrl}} (not a hard-coded host).
  4. Update the environment defaults: baseUrl = https://api.parseflow.tech.

Public doc link (update after re-import): Postman Documenter

Troubleshooting​

ProblemFix
401 UnauthorizedSet API_KEY to a valid pfk_* key
Trial key not receivedCheck spam; wait 2–3 minutes; try a non-disposable email
Connection errorConfirm GET /health works in a browser
413 / payload too largeReduce file size or split into smaller documents

See also POSTMAN_SETUP.md for the full walkthrough.
