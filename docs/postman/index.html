Parseflow Postman Quickstart

This guide walks you through using Parseflow with Postman step by step, from importing the API to downloading the ZIP response and inspecting the main output files.

Base URL for the live API: https://parseflow-api.thankfulisland-fec0f8f2.westus2.azurecontainerapps.io

1) Install Postman​

  1. Open Postman.
  2. If you do not already have it, download it from https://www.postman.com/downloads/.

2) Create a Postman environment​

  1. Click the gear icon in the top right and choose Manage Environments.
  2. Click Add to create a new environment.
  3. Name it Parseflow Live.
  4. Add these variables:
    • baseUrl = https://parseflow-api.thankfulisland-fec0f8f2.westus2.azurecontainerapps.io
    • apiKey = (leave blank for now)
  5. Click Save.
  6. Select the Parseflow Live environment from the environment dropdown in the top right.

3) Create the collection manually (no import)​

If you want to set up Postman from scratch, do it manually without importing any collection file.

  1. Click New in the top left.
  2. Choose Collection.
  3. Name the collection Parseflow API.
  4. Click Create.

Add the health request​

  1. Click the new collection and choose Add a request.
  2. Name it Health.
  3. Set the method to GET.
  4. Enter the URL: {{baseUrl}}/health.
  5. Click Save.

Add the API key issuance request​

  1. In the collection, click Add a request.
  2. Name it Issue API Key.
  3. Set the method to POST.
  4. Enter the URL: {{baseUrl}}/v2/admin/api-keys.
  5. In the Body tab, choose raw and JSON.
  6. Enter: 
    {
      "name": "postman-test"
    }
  7. Click Save.

Add the process file request​

  1. Add another request and name it Process File.
  2. Set the method to POST.
  3. Set the URL to {{baseUrl}}/v2/process.
  4. Open the Body tab and choose form-data.
  5. Add a key named file, change its type to File, and attach a PDF or DOCX.
  6. Add optional keys like mode = deterministic and preset = rag-optimized.
  7. Click Save.

Add the process text request​

  1. Add another request and name it Process Text.
  2. Set the method to POST.
  3. Use URL {{baseUrl}}/v2/process.
  4. In Body, choose form-data.
  5. Add a key text with a sample value, for example Invoice total $125 due Friday.
  6. Optionally add mode = deterministic.
  7. Click Save.

Add the analyze request​

  1. Add another request and name it Analyze.
  2. Set the method to POST.
  3. Use URL {{baseUrl}}/v2/analyze.
  4. In Body, choose form-data.
  5. Add text with a sample value.
  6. Click Save.

4) Confirm the health endpoint works​

  1. In the Parseflow API collection, open Health.
  2. Click Send.
  3. You should see a response like: 
    {
      "status": "ok",
      "version": "0.2.0"
    }

If this works, Postman is connected to the live Parseflow API.

5) Create or set your API key​

If you already have a Parseflow API key, paste it into the apiKey environment variable.

If you need a new key and the admin endpoint is available, do this:

  1. Open Issue API Key.
  2. Click Send.
  3. Copy the returned API key.
  4. In the environment editor, paste it into apiKey.

If the admin endpoint is protected, add the required header(s) under Headers, for example:

  • X-Admin-Secret: <your-admin-secret>

6) Set default auth headers​

  1. Click the collection name (Parseflow API).
  2. Choose Edit.
  3. Open the Authorization tab.
  4. Select API Key.
  5. Set the Key to X-API-Key.
  6. Set the Value to {{apiKey}}.
  7. Set Add to to Header.
  8. Click Update.

This makes the API key send automatically for every request in the collection.

7) Process a file and download the ZIP​

  1. Open the Process File request.
  2. Click Send.
  3. If the response is binary, click Save Response.
  4. Save the response as parseflow-output.zip.

8) Open the ZIP and inspect the files​

Unzip the downloaded ZIP file and inspect the main outputs:

  • result.json
  • document.md
  • chunks.jsonl

result.json is the main structured parse output. document.md is the cleaned text and markdown export. chunks.jsonl contains one JSON object per chunk, which is the best format for indexing, embeddings, or search.

9) Inspect a chunk​

  1. Open chunks.jsonl in a text editor.
  2. Each line is a separate JSON object.
  3. Look for fields such as id, text, start, end, and metadata.
  4. Use that chunk text in your downstream application.

10) If you want, you can still import later​

If you decide to use the OpenAPI import or a ready-made collection file later, the API URLs and auth are already set up in your environment.

11) Troubleshooting​

  • 401 Unauthorized → your apiKey is missing or invalid.
  • no response / timeout → check {{baseUrl}} and make sure GET /health succeeds.
  • ZIP response fails to open → save it as .zip and unzip locally.

4) Confirm the health endpoint works​

  1. In the imported collection, find GET /health.
  2. Click the request.
  3. Click Send.
  4. You should see a response like: 
    {
      "status": "ok",
      "version": "0.2.0"
    }

If this works, Postman is talking to the live Parseflow service.

5) Create or set your API key​

If you already have a Parseflow API key, paste it into the apiKey environment variable.

If you need a new key and the admin endpoint is available, do this:

  1. Create a new request in the collection.
  2. Method: POST
  3. URL: {{baseUrl}}/v2/admin/api-keys
  4. Headers:
    • Content-Type: application/json
  5. Body type: raw and JSON: 
    {
      "name": "postman-test"
    }
  6. Click Send.

If your admin endpoint is protected, you may also need an admin header such as X-Admin-Secret or an equivalent. Set that in the Headers tab.

When the request succeeds, copy the returned API key into your environment:

  • apiKey = pfk_...

6) Set default auth headers​

  1. Open the collection settings.
  2. Go to Authorization.
  3. Choose API Key.
  4. Set the Key to X-API-Key.
  5. Set the Value to {{apiKey}}.
  6. Set Add to to Header.
  7. Save.

This makes sure every request in the collection sends your key automatically.

7) Process a file and download the ZIP​

  1. Find the request POST /v2/process in the collection.
  2. Click it.
  3. In the Body tab, choose form-data.
  4. Add these fields:
    • file → change the type from Text to File, then choose a PDF, DOCX, or TXT file from your computer.
    • mode → deterministic (optional)
    • preset → rag-optimized or extraction-focused (optional)
    • zip_name → parseflow-output (optional)
  5. Click Send.

What you should see​

  • The response should return a ZIP file.
  • Postman may show a binary response.
  • Click Save Response.
  • Save the file as parseflow-output.zip.

8) Open the ZIP and view the key files​

Extract the ZIP file and inspect these files:

  • result.json
    • Contains the parsed structure and the main extraction output.
  • document.md
    • Contains the cleaned text and markdown version of the document.
  • chunks.jsonl
    • Contains one JSON object per chunk.
    • Each line is a chunk, so you can stream it into search, embeddings, or RAG pipelines.

Why this matters​

  • result.json is the main machine-readable output.
  • document.md is easy to read and useful for review or publishing.
  • chunks.jsonl is the best format for downstream use in search, vectorization, or summary pipelines.

9) Inspect a chunk, step-by-step​

  1. Open chunks.jsonl in a text editor.
  2. Each line is a complete JSON object.
  3. Look for fields like:
    • id
    • text
    • start / end
    • any metadata your pipeline includes
  4. Use the chunk text for embedding, search indexing, or chaining into a summarization flow.

10) Quick Postman checks​

  • If GET /health fails, the service URL is wrong or the app is down.
  • If POST /v2/process fails with 401, your apiKey is wrong.
  • If the response is binary but not a ZIP, check the request body and ensure file is attached correctly.

11) Example walkthrough​

  1. Import the collection.
  2. Set baseUrl and apiKey.
  3. Run GET /health.
  4. Issue a key if needed via POST /v2/admin/api-keys.
  5. Run POST /v2/process with a file.
  6. Save the ZIP and open chunks.jsonl.

12) Notes for users​

  • The public API URL does not change unless you redeploy to a new host.
  • You do not have to store the ZIP file in Postman.
  • The ZIP is the “main output bundle” that contains everything useful from the parse.
