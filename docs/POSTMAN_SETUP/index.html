Postman Setup Guide for Parseflow API

This guide walks you through importing and using the public Parseflow Postman collection.

Quick Start​

1. Import the collection​

  • Option A (Easiest): Download the parseflow-demo-public.postman_collection.json from the repo

    • In Postman: File → Import → Upload the JSON file

  • Option B: Copy/paste the collection JSON link into Postman

2. Create the environment​

In Postman, create a new Environment named Parseflow Demo with these variables:

VariableValueExample
baseUrlYour Parseflow API base URLhttps://parseflow-api.thankfulisland-fec0f8f2.westus2.azurecontainerapps.io
API_KEYDemo API key (provided)pk_demo_...
API_KEY_HEADER_NAMEHeader key for API keyX-API-Key

Important: Do NOT add any admin secrets or admin-only headers to the public environment.

3. Verify the environment​

  • In Postman, select Parseflow Demo from the Environment dropdown
  • Hover over {{baseUrl}} in a request to confirm it resolves correctly

Collection Structure​

Public Folder​

These requests are safe to share and publish.

Health (GET)​

Verify the API is running.

Request:

GET {{baseUrl}}/health

No auth required.

Expected response:

{
  "status": "ok",
  "version": "0.2.0"
}

Process (file upload) (POST)​

Upload a PDF, DOCX, or TXT file for processing.

Request:

POST {{baseUrl}}/v2/process

Headers:

X-API-Key: {{API_KEY}}

Body (form-data):

KeyTypeValueNotes
filefile(select file)PDF, DOCX, or TXT file
analyzetext(optional)Short text to bias analysis, e.g. "Invoice total $125 due Friday"
modetext(optional)Parsing mode, e.g. "strict"
presettext(optional)Export preset, e.g. "invoice"
zip_nametext(optional)Custom ZIP filename

Saving the response:

  • Use Postman "Send and Download" button → choose filename (e.g., parseflow-output.zip)
  • Or: Send → right-click response → "Save Response" → "Save to file"

Expected response:

  • HTTP 200 with Content-Type: application/zip
  • Binary ZIP file containing:
    • result.json — structured parse output
    • document.md — flattened markdown
    • chunks.jsonl — line-delimited JSON chunks
    • Optional exporter artifacts

Process (text-only analyze) (POST)​

Quick analysis of raw text without uploading a file.

Request:

POST {{baseUrl}}/v2/process

Headers:

X-API-Key: {{API_KEY}}

Body (form-data):

KeyTypeValueNotes
texttextInvoice total $1,250.00. Due date: 2026-06-01. Customer: ACME Corp.Text-only input, no file required

Expected response:

  • JSON object or ZIP depending on the preset

Note: This endpoint expects form-data input, not raw JSON. Use the text form field in Postman.

Common Tasks​

Task 1: Verify the API is live​

  1. Select Parseflow Demo environment
  2. Click Health request
  3. Click "Send"
  4. Look for status code 200 and response {"status":"ok","version":"0.2.0"}

Task 2: Upload and process a document​

  1. Ensure Parseflow Demo environment is selected
  2. Click Process (file upload) request
  3. In Body tab, form-data:
    • Click the file field → click file icon → select a PDF or DOCX from your computer
    • (Optional) Add text or analyze text, e.g., "Invoice total $125 due Friday"
  4. Click "Send and Download"
  5. Save as parseflow-output.zip
  6. Open the ZIP and inspect result.json and document.md

Task 3: Analyze text without file upload​

  1. Click Process (text-only) request
  2. In Body tab, form-data:
    • Add field text
    • Value: Invoice total $125 due Friday
  3. Click "Send"
  4. Review the response for extracted structure

Task 4: Inspect key and usage data​

  1. Click Key Info request

  2. Click "Send"

  3. Confirm the response includes key_id, label, and plan

  4. Click Usage Info request

  5. Click "Send"

  6. Confirm the response includes current_usage, monthly_limit, and remaining

  7. Click Process (text-only analyze) request

  8. In Body tab, edit the JSON text field

  9. Click "Send"

  10. Check the response for extracted structure

Rate Limits & Demo Key Info​

The demo API key is:

  • Limited to 100 requests per hour (adjust based on quotas)
  • Expires in 7 days (operator will rotate)
  • Request body limit: 10 MB per file
  • Response timeout: 60 seconds

If you hit the rate limit, you'll receive a 429 response:

{
  "detail": "Rate limit exceeded. Max 100 requests per hour."
}

Troubleshooting​

"UNAUTHORIZED: invalid API key"​

  • Confirm X-API-Key header is set
  • Confirm the header value matches the API_KEY variable
  • Confirm the demo key hasn't expired (7-day TTL)

"404 Not Found"​

  • Confirm {{baseUrl}} resolves (check environment variables)
  • Confirm the URL doesn't have trailing slashes

"Process request returns an error or HTML"​

  • Check the file format (must be PDF, DOCX, or TXT)
  • Check file size (< 10 MB)
  • Check Content-Type headers are correct
  • Review response body for detailed error message

"ZIP download fails or file is corrupted"​

  • Use Postman "Send and Download" instead of copying the response
  • Ensure the Content-Type header is application/zip

Sample Files to Test​

If you don't have a test file handy:

  1. Create a sample invoice (TXT):

    INVOICE #2026-001
    Date: 2026-05-25
    Customer: ACME Corp
    Total: $1,250.00
    Due: 2026-06-01
    Payment terms: Net 30

  2. Create a sample contract (TXT):

    CONTRACT TERMS
    Parties: Company A and Company B
    Effective Date: 2026-05-25
    Payment: $500/month
    Term: 12 months
    Renewal: Auto-renew unless notice given 30 days prior

  3. Save as .txt and upload via the Process (file upload) request

Exporting for Teams​

To share the collection with teammates:

  1. In Postman: Collections → Parseflow Demo → ... → Export
  2. Choose format: Collection v2.1 (recommended)
  3. Save the JSON file
  4. Share the JSON with teammates
  5. They import via File → Import → Upload JSON

Important: Ensure you don't export the Operator folder or any admin secrets.
