API

Use this page as the quick implementation reference. For the full contract, see API Reference.

Core flow​

The API supports two layers:

  • v1 for straightforward parse-and-pack workflows
  • v2 for structured extraction, jobs, search, stats, and API-key flows

Base URLs​

Public gateway:

https://jsonparser-production-f9af.up.railway.app

Key headers​

  • X-API-Key - public API authentication header
  • X-Client-Id - optional label for your own usage tracking
  • Idempotency-Key - prevents duplicate v2 process requests
  • X-BYOK-Key - provider key for BYOK advanced routes

Most used endpoints​

POST /process​

Simple parse and chunk flow.

POST /pack​

Same processing flow, but returns a ZIP containing:

  • results.json
  • document.md
  • chunks.jsonl

POST /v2/process​

Structured extraction, preset selection, schema support, and analysis.

POST /v2/analyze​

Chunking analysis without the extraction payload.

POST /v2/batch​

Bulk document processing.

POST /v2/index and GET /v2/search​

Persist and search indexed documents.

POST /v2/jobs​

Async job creation with optional callback delivery.

Quick request shape​

Send one input:

nametyperequired
filefileno
textstringno

Optional controls:

nametyperequirednotes
chunk_sizeintnodefaults to 2000
overlapintnodefaults to 200
modestringnodeterministic or byok_assisted (legacy aliases still accepted where supported)
presetstringnov2 process/analyze only
schema_jsonstringnov2 process/index only
enforce_schemaboolnov2 process only

Default response fields​

  • request_id
  • document
  • result
  • metadata_summary

v2 adds:

  • api_version
  • preset
  • extraction
  • analysis

Example request​

curl -X POST https://jsonparser-production-f9af.up.railway.app/v2/process \
  -H "X-API-Key: YOUR_PARSEFLOW_API_KEY" \
  -H "X-Client-Id: docs-preview" \
  -H "Idempotency-Key: demo-001" \
  -F "text=Company: Acme Inc\nDate: 2025-01-15\nAmount: $5000" \
  -F "mode=deterministic" \
  -F "preset=extraction-focused"

Error behavior​

Common status codes:

codemeaning
400missing input, invalid preset, invalid schema JSON
401missing or invalid authentication header
413upload too large
422schema validation failed
504processing timed out
