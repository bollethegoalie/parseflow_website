Deployment & Launch Checklist

This is the complete checklist for ensuring Parseflow is production-ready before full public launch.

Pre-Launch Security & Verification (✅ DONE)​

  • Stripe configured in live mode (not test mode)
  • Webhook endpoint active and receiving events
  • API key issuance working via webhook
  • Test checkout completed successfully
  • Documentation updated for live mode (no test/sandbox notices)
  • Announcement bar removed
  • Contact page published
  • Pricing page with buy links live
  • All pages verified for outdated content

Infrastructure & Backend​

  • CRITICAL: Restore DOCFLOW_STRIPE_WEBHOOK_SECRET in Railway to live whsec_... value

    • This re-enables webhook signature verification
    • Essential before accepting real payments
    • Location: Railway → Project → Environment Variables
    • Redeploy after setting

  • Verify Railway deployment healthy (green status)

  • Check Railway logs for any errors

  • Confirm API response times < 5 seconds

  • Set up monitoring/alerting (optional: UptimeRobot, Sentry)

  • Backup strategy in place (if using database)

  • Rate limiting configured (optional: via Railway or API Gateway)

Documentation Quality​

  • No broken internal links (verified)
  • All examples tested and working
  • API Reference complete and accurate
  • Getting Started walkthrough clear
  • Contact page with support info
  • Tutorials with real-world examples
  • Advanced Examples for developers
  • FAQ covers common questions
  • Spelling/grammar check (recommended: run through tool)
  • Mobile responsiveness verified
  • Dark mode tested (if applicable)

Website & SEO​

  • Sitemap generated and submitted to Google Search Console
  • robots.txt in place
  • Meta descriptions on all pages
  • Open Graph tags for social sharing
  • Custom domain verified (docs.parseflow.tech)
  • SSL certificate valid (should be automatic with GitHub Pages + custom domain)
  • Google Analytics or equivalent set up
  • Google Search Console configured

Payment & Billing​

  • Stripe webhook endpoint verified in Dashboard
  • Webhook events being logged
  • Test checkout completes end-to-end
  • Email key delivery working
  • Stripe account verified (no restrictions)
  • Tax settings configured (if applicable)
  • Billing page has correct payment links
  • Support email addresses working

External Services & Links​

  • No public GitHub links to private repos
  • No public Postman collection links
  • No internal/staging URLs in docs
  • All payment links go to live Stripe (not test)
  • Webhook URL in docs is production endpoint
  • Contact email monitored and responding

Code & Repository​

  • No sensitive data in public repos (API keys, secrets, etc.)
  • .env.example file present (no real secrets)
  • Git history cleaned (remove any accidentally committed keys)
  • README updated with deployment instructions
  • License file present
  • No verbose debug logging in production
  • Error messages don't leak sensitive info

Launch Communications​

  • Announcement text prepared
  • Social media posts drafted
  • Early access/beta testers notified (if applicable)
  • Founder story / positioning ready
  • Share locations identified (Reddit, SaaSHub, Quora, HN, etc.)

Post-Launch Monitoring​

  • Check for errors in logs during first 24 hours
  • Monitor API response times
  • Watch for unexpected webhook failures
  • Monitor Stripe dashboard for webhook issues
  • Respond to early support requests quickly
  • Track conversion rate through checkout
  • Collect user feedback

First 30 Days​

  • Revenue, cost, and profit tracking spreadsheet
  • Analytics dashboard set up
  • Conversion metrics identified
  • Early user feedback collected
  • Document what's working and what's not
  • Plan pricing adjustments if needed
  • Plan next feature/improvement iteration

Immediate Next Steps​

1. Restore Webhook Secret (HIGH PRIORITY)​

# In Railway dashboard:
# Project → Settings → Environment Variables
# Set: DOCFLOW_STRIPE_WEBHOOK_SECRET = whsec_live_...

# Then: Click Deploy to restart the service

Why: Without this, webhook events from real Stripe checkouts won't be signature-verified. This is a critical security requirement.

2. Set Up Basic Monitoring (Recommended)​

# Option A: UptimeRobot (free tier)
# Add monitor for: https://jsonparser-production-f9af.up.railway.app/health
# Check every 5 minutes
# Alert on down/slow responses

# Option B: CloudFlare Analytics (included with custom domain)
# Already enabled, check analytics at domain registrar

3. Add Sitemap & robots.txt​

These files help search engines discover and index your content.

robots.txt (should be at docflow-docs/static/robots.txt):

User-agent: *
Allow: /
Disallow: /admin/

Sitemap: https://docs.parseflow.tech/sitemap.xml

sitemap.xml - Docusaurus generates this automatically; verify at:

https://docs.parseflow.tech/sitemap.xml

4. Submit to Search Engines​

  • Google Search Console — Submit sitemap, verify domain
  • Bing Webmaster Tools — Similar process
  • These take 1-2 weeks to fully crawl

5. Create Analytics Spreadsheet​

Track from day 1:

  • Daily traffic (from Cloudflare or Google Analytics)
  • Checkout conversions
  • API requests per customer
  • Estimated revenue vs. costs
  • Customer acquisition cost

6. Prepare Marketing Content​

  • Write 1-2 paragraph product description
  • Prepare 3-5 founder story / origin points
  • Identify 5-10 communities where you'd share (Reddit, SaaS forums, etc.)
  • Create 2-3 social media posts
  • Draft FAQ for early questions

Success Criteria​

You're ready to launch fully when:

✅ Backend is secure (webhook secret restored)
✅ Docs are complete and accurate
✅ Payment flow works end-to-end
✅ Support email is monitored
✅ Monitoring is in place
✅ You can handle early support requests

Contingency: If Something Breaks​

API not responding?

  • Check Railway logs
  • Verify environment variables
  • Check Stripe webhook status
  • Try redeploying

Payment not working?

  • Check Stripe webhook endpoint in Dashboard
  • Verify webhook signing secret
  • Check Railway logs for webhook delivery
  • Verify API key generation code

Users can't find docs?

  • Verify GitHub Pages DNS
  • Check custom domain CNAME record
  • Clear browser cache
  • Try incognito mode

No traffic after launch?

  • Check search console for indexing
  • Verify analytics tags are installed
  • Share directly with early users first
  • Post on relevant communities
