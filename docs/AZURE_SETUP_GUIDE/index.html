Azure Setup Guide: Parseflow Deployment & Demo Key

This guide walks you through:

  1. Finding your Azure resource details (subscription, resource group, app name)
  2. Generating and setting the admin secret safely
  3. Creating a rate-limited demo API key for public use
  4. Verifying everything before publishing

Important: This guide avoids using azd deploy (which has been blocked in your subscription). We use direct Azure CLI commands instead.

Part 1: Find Your Azure Resource Details​

Step 1a: Find your Azure Subscription ID​

Via Azure Portal (easiest):

  1. Go to https://portal.azure.com
  2. Search for "Subscriptions" in the search bar
  3. Click on your subscription name
  4. Copy the Subscription ID (e.g., 12345678-1234-1234-1234-123456789abc)

Via Azure CLI:

az account list --output table

Copy the SubscriptionId column value for the subscription you use.

Store it locally (for reference):

SUBSCRIPTION_ID="your-subscription-id-here"
echo "Subscription: $SUBSCRIPTION_ID"

Step 1b: Find your Resource Group name​

Via Azure Portal:

  1. Go to https://portal.azure.com
  2. Search for "Resource groups"
  3. Click your resource group (likely named parseflow-*-rg)
  4. Copy the resource group name

Via Azure CLI:

az group list --output table

Look for a group with name like parseflow-*-rg or similar.

Store it locally:

RESOURCE_GROUP="your-resource-group-name"
echo "Resource Group: $RESOURCE_GROUP"

Step 1c: Find your Container App name​

Via Azure Portal:

  1. Go to https://portal.azure.com
  2. Search for "Container Apps"
  3. Click on your Container App (likely named parseflow-api or similar)
  4. Copy the name from the Overview page

Via Azure CLI:

az containerapp list --resource-group "$RESOURCE_GROUP" --output table

Copy the Name column value.

Store it locally:

CONTAINER_APP_NAME="your-app-name"
echo "Container App: $CONTAINER_APP_NAME"

Step 1d: Verify all three values are correct​

SUBSCRIPTION_ID="your-subscription-id"
RESOURCE_GROUP="your-resource-group"
CONTAINER_APP_NAME="your-app-name"

echo "Subscription: $SUBSCRIPTION_ID"
echo "Resource Group: $RESOURCE_GROUP"
echo "App Name: $CONTAINER_APP_NAME"

All three should be non-empty before proceeding.

Part 2: Generate & Set the Admin Secret​

Step 2a: Generate a strong admin secret locally​

ADMIN_SECRET=$(openssl rand -hex 32)
echo "Generated Admin Secret: $ADMIN_SECRET"
echo "Save this somewhere secure (1Password, LastPass, etc.)"

Important: This secret must be kept private. Never commit it to git or share publicly.

Step 2b: Verify you're authenticated to Azure​

az account show --output table

This should show your subscription and account. If it fails, run:

az login
az account set --subscription "$SUBSCRIPTION_ID"

Step 2c: Set the admin secret in your Container App​

SUBSCRIPTION_ID="your-subscription-id"
RESOURCE_GROUP="your-resource-group"
CONTAINER_APP_NAME="your-app-name"
ADMIN_SECRET="your-generated-secret"

az containerapp update \
  --name "$CONTAINER_APP_NAME" \
  --resource-group "$RESOURCE_GROUP" \
  --set-env-vars DOCFLOW_API_KEY_ADMIN_SECRET="$ADMIN_SECRET"

Expected output:

  • You should see a JSON response with the app configuration
  • Look for "DOCFLOW_API_KEY_ADMIN_SECRET" in the properties.template.containers[0].env section

Verify it was set:

az containerapp show \
  --name "$CONTAINER_APP_NAME" \
  --resource-group "$RESOURCE_GROUP" \
  --output json | grep -i "admin_secret"

You should see output containing your secret. The app will restart automatically (this takes ~30 seconds).

Step 2d: Wait for the app to restart​

# Wait 30 seconds for restart
sleep 30

# Verify the app is still healthy
APP_URL=$(az containerapp show --name "$CONTAINER_APP_NAME" --resource-group "$RESOURCE_GROUP" --query properties.configuration.ingress.fqdn -o tsv)
echo "App URL: https://$APP_URL"

curl -sS "https://$APP_URL/health" | jq .

Expected response:

{
  "status": "ok",
  "version": "0.2.0"
}

If this fails, wait another 30 seconds and try again. If it still fails, check the Container App logs in Azure Portal.

Part 3: Create a Rate-Limited Demo API Key​

Step 3a: Create the key via the admin endpoint​

ADMIN_SECRET="your-admin-secret"
APP_URL="https://your-app-url.azurecontainerapps.io"

# Create a demo key with 1-hour TTL
curl -sS -X POST "$APP_URL/v2/admin/api-keys" \
  -H "Content-Type: application/json" \
  -H "X-Docflow-Admin-Secret: $ADMIN_SECRET" \
  -d '{
    "name": "public-demo",
    "scopes": ["process"],
    "expires_in_days": 0.04167,
    "rate_limit_per_hour": 100
  }' | jq .

If you get "invalid admin secret" error:

  • Double-check that ADMIN_SECRET exactly matches what you set in Step 2c
  • Wait 30 more seconds for the app restart to fully complete
  • Try again

Expected response (on success):

{
  "api_key": "pk_demo_a1b2c3d4e5f6...",
  "name": "public-demo",
  "expires_at": "2026-05-25T14:30:00Z",
  "scopes": ["process"]
}

Copy the api_key value. This is your demo key.

Step 3b: Store the demo key securely​

DEMO_API_KEY="pk_demo_a1b2c3d4e5f6..."
echo "Demo API Key: $DEMO_API_KEY"
echo "Keep this in a secure location (1Password, or as a Postman environment variable)"

Step 3c: Test the demo key​

DEMO_API_KEY="your-demo-key"
APP_URL="https://your-app-url.azurecontainerapps.io"

# Test with health endpoint
curl -sS "$APP_URL/health" | jq .

# Test with process endpoint (text-only)
curl -sS -X POST "$APP_URL/v2/process" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $DEMO_API_KEY" \
  -d '{"text": "Invoice total $125 due Friday"}' | jq .

Both should return 200 status.

Part 4: Verify Safety Before Publishing​

Checklist before publishing demo key publicly​

  • Admin secret is set in Azure and NOT exposed
  • Demo key has short TTL (1 hour recommended, 7 days max)
  • Demo key has rate limit (100-1000 requests/hour recommended)
  • Demo key has limited scopes (only "process")
  • Operator contact info is documented (for abuse reports)
  • Monitoring/alerting is configured in Azure (see Part 5)
  • API is reachable and healthy

Run this final verification:

APP_URL="https://your-app-url.azurecontainerapps.io"

# 1. Health
echo "=== Health Check ==="
curl -sS "$APP_URL/health" | jq .

# 2. Test demo key (replace with your key)
DEMO_API_KEY="pk_demo_..."
echo "=== Process Test (Demo Key) ==="
curl -sS -X POST "$APP_URL/v2/process" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $DEMO_API_KEY" \
  -d '{"text":"Sample test"}' | head -c 100

# 3. Verify admin endpoint is protected
echo -e "\n=== Admin Protection Check ==="
curl -sS -X POST "$APP_URL/v2/admin/api-keys" \
  -H "Content-Type: application/json" \
  -H "X-Docflow-Admin-Secret: wrong-secret" \
  -d '{"name":"test"}' | jq .
# Should return 401 UNAUTHORIZED

All three should work correctly.

Part 5: Configure Monitoring & Rate Limits (Optional but Recommended)​

5a: Enable Application Insights​

Via Azure Portal:

  1. Go to your Container App
  2. Click "Monitoring" → "Application Insights"
  3. Click "Enable" if not already enabled
  4. Save configuration

Via Azure CLI:

# If not already set
az containerapp update \
  --name "$CONTAINER_APP_NAME" \
  --resource-group "$RESOURCE_GROUP" \
  --enable-application-insights

5b: Set up alerts for API errors​

Via Azure Portal:

  1. Go to Application Insights for your app
  2. Click "Alerts" → "Create alert rule"
  3. Condition: "Failed requests" > 10 in 5 minutes
  4. Action: Email yourself or Slack webhook

5c: Track demo key usage​

In your Postman Docs or README, document:

  • Current rate limit: 100 requests/hour
  • Quota reset: Every hour
  • Support contact for quota increase

Part 7: Key Rotation & Maintenance​

Rotate the demo key every day (recommended)​

ADMIN_SECRET="your-admin-secret"
APP_URL="https://your-app-url.azurecontainerapps.io"

# Revoke old key (if your API supports DELETE /v2/admin/api-keys/{id})
# curl -X DELETE "$APP_URL/v2/admin/api-keys/old-key-id" \
#   -H "X-Docflow-Admin-Secret: $ADMIN_SECRET"

# Create new key
curl -sS -X POST "$APP_URL/v2/admin/api-keys" \
  -H "Content-Type: application/json" \
  -H "X-Docflow-Admin-Secret: $ADMIN_SECRET" \
  -d '{"name":"public-demo","scopes":["process"],"expires_in_days":0.04167}' | jq .

Update Postman and README with the new key.

If demo key is compromised​

  1. Immediately revoke the key (if supported)
  2. Generate a new secret: 
    NEW_ADMIN_SECRET=$(openssl rand -hex 32)
  3. Update Azure Container App with new secret (Step 2c)
  4. Create a new demo key with the new admin secret
  5. Update all public documentation

Troubleshooting​

Error: "invalid admin secret"​

  • Confirm the ADMIN_SECRET value matches what you set in Azure
  • Wait 30+ seconds after setting the env var for the app to restart
  • Try the health check endpoint first to confirm the app is running

Error: "Rate limit exceeded"​

  • Demo key is limited to 100 req/hour by default
  • Wait 1 hour for quota to reset
  • Or request a higher quota key (create a new key with higher limit)

Health check fails​

  • Confirm the Container App is running (Azure Portal → Container Apps → your app)
  • Check Container App logs (Monitoring → Log stream)
  • Confirm firewall/networking allows inbound traffic

App doesn't restart after setting env var​

  • Azure sometimes requires 30-60 seconds
  • Check Container App status in portal
  • If stuck, manually restart via portal or CLI: 
    az containerapp revision restart --name "$CONTAINER_APP_NAME" --resource-group "$RESOURCE_GROUP"
